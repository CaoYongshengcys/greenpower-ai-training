# AI Agent 实战工作坊简介

**主题**：驾驭 AI Agent —— Prompt · Context · Harness 三要素实战

**面向对象**：中欧商学院 MBA 学员（偏产业与金融背景）

**总时长**：全天约 7 小时

**核心工具**：Claude Code + CC Switch + 阿里百炼

**核心理念**：Agent 是杠杆，不是替代；驾驭 Agent 的关键在于三要素——设计好任务（Prompt）、管理好信息（Context）、设定好规则（Harness）

---

## 上午：认知构建与工具演示（3.5 小时）

- **破冰导入**：理解 AI Agent 与传统 ChatGPT 的本质区别——Agent 能多步骤自主执行、读写文件、运行代码，而非简单的一问一答
- **三要素精讲**：
  - **Prompt**：结构化任务指令书（CO-STAR + Constraint 框架），现场对比"烂 Prompt"与"好 Prompt"的输出差异
  - **Context**：四种上下文类型（外部文件、对话历史、系统配置、工具能力），核心是管理 Agent 的"视野"与"记忆"
  - **Harness**：四类约束护栏（格式约束、思维链约束、边界约束、校验机制），防止 Agent 幻觉与跑偏
- **工具链演示**：Claude Code（执行引擎）+ CC Switch（配置中枢）+ 阿里百炼（模型供给）的协同使用
- **两个完整案例**：市场调研与财报解读，重点展示"Agent 犯错 → 人工纠正"的迭代价值

## 下午：小组实战与汇报（3.5 小时）

- **8 组 × 5 人**，从 13 个场景任务卡中选取一个进行 90 分钟实战，场景覆盖：
  - **金融类**：财报解读与同业对比、IC Memo 草稿生成、舆情监控简报、投资组合月报
  - **产业类**：产业链图谱绘制、商业模式画布生成、政策影响评估、竞品功能对比矩阵
  - **通用类**：会议纪要智能整理、数据故事化汇报、商业计划书大纲、行业术语词典、调研报告生成
- 实战流程：Prompt 设计 → Agent 执行 → 人工校验 → 迭代优化，鼓励"翻车"——翻车是最好的学习材料
- 每组 5-6 分钟汇报：展示最终成果、Prompt 迭代记录、翻车记录与心得总结

---

## 核心带走

1. **Agent 是杠杆，不是替代**——它放大执行力，不替代判断力
2. **质量 80% 取决于 Prompt，15% 取决于 Context，5% 取决于模型**——不要迷信更贵的模型，先优化你的指令
3. **永远假设 Agent 会犯错**——边界约束、校验机制、人工复核，三道防线缺一不可
