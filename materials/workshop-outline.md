### 09:00-09:20 破冰与导入（20分钟）

#### 开场白建议

> "大家好，我是来自上海交大人工智能研究院的[姓名]。今天我们不讲算法，不讲代码，讲的是——**怎么让 AI 像你的实习生一样帮你干活**。
>
> 在座各位都是产业界和金融界的中坚力量。你们的时间很贵，但每天可能有 30%-50% 花在信息整理、材料撰写、数据搬运这些重复性工作上。今天的工作坊，目标就是让你们带走一个认知：**Agent 不是来替代你的判断力的，而是来放大你的执行力的。**"

#### 三个关键对比（PPT建议）

| 对比维度 | 传统 ChatGPT | AI Agent（Claude Code） |
|---------|------------|----------------------|
| 交互方式 | 一问一答，人驱动 | 多步骤自主执行，人设定目标 |
| 能力边界 | 只动嘴（生成文字） | 能动手（读写文件、运行代码、调用工具） |
| 使用门槛 | 需要人持续输入 | 一次设定，自主完成复杂任务链 |
| 错误模式 | 说错话（幻觉） | 做错事（文件覆盖、数据误读），需人工校验 |

#### 今日承诺

> "今天结束时，我希望每个人都能回答三个问题：
> 1. Agent 能帮我做什么？（能力边界）
> 2. Agent 不能帮我做什么？（能力盲区）
> 3. 我怎么让它做得更好？（驾驭方法）"

#### 课堂纪律

- 上午请跟随演示操作，有问题随时举手
- 下午小组实战，每组需要一个"记录员"记录迭代过程
- 手机静音，但鼓励拍照记录关键屏幕

### 09:20-09:50 三要素讲解：Prompt（30分钟）

#### 概念引入（5分钟）

**类比**：
> "Prompt 就是你给下属布置工作时说的话。但大多数人对 AI 的说话方式，还停留在'搜索引擎时代'——输入关键词，期待一个完美答案。对 Agent 来说，这远远不够。"

**定义**：
> "Prompt 是一份**结构化的任务指令书**。它告诉 Agent：**你要做什么、给谁看、做到什么程度、输出什么格式、不能踩哪些红线**。

#### Prompt 结构拆解（10分钟）

推荐教学框架（简化版 CO-STAR）：

| 要素 | 英文 | 含义 | 示例 |
|------|------|------|------|
| **C**ontext | 背景 | 这个任务的前因后果 | "我们是某私募基金的投研团队，正在评估宁德时代2024年Q3的投资价值" |
| **O**bjective | 目标 | 具体要达成什么 | "提取财报中的5个核心财务指标，并与比亚迪、LG新能源进行横向对比" |
| **S**tyle | 风格 | 输出的语气和专业程度 | "用资深分析师的口吻，数据驱动，避免主观臆断" |
| **T**one | 语调 | 情感色彩 | "客观、冷静、略带审慎" |
| **A**udience | 受众 | 给谁看 | "面向投资委员会，他们熟悉财务术语但时间有限" |
| **R**esponse | 格式 | 输出形式 | "Markdown表格呈现对比数据，最后附3条投资亮点和3条风险提示" |

#### 现场演示（15分钟）

**演示任务**：让 Agent 分析一句话的公司描述

**步骤1：展示"烂Prompt"**

```
帮我分析一下这家公司。
```

**预期结果**：Agent 输出泛泛而谈，没有聚焦，没有结构。

> "大家看，Agent 确实给出了一些信息，但像一份 Wikipedia 摘要，对我们的投资决策没有直接帮助。问题出在：我们没有给它**目标、受众和格式约束**。

**步骤2：展示"好Prompt"**

```
背景：我们是某私募基金投研团队，正在评估新能源汽车电池赛道的头部企业。

任务：基于公开信息，对宁德时代进行商业模式分析。

要求：
1. 用商业模式画布（Business Model Canvas）的9个模块进行结构化呈现
2. 每个模块至少列出2个关键要点
3. 对不确定的信息，标注[推测]
4. 最后附一段200字的投资价值摘要，面向投资委员会
5. 输出格式：Markdown表格 + 文字段落

约束：只基于公开可验证的信息，不要编造未公开的财务数据或合作细节。
```

**预期结果**：输出结构清晰、模块完整、有标注、有摘要。

> "对比这两个输出，你们看到了什么？第二个 Prompt 字数是第一个的10倍，但产出的价值可能是100倍。这不是因为 Agent 变聪明了，而是因为我们给了它**跑道**。

#### 教学要点总结

- **Prompt 的长度 ≠ 复杂度**：关键是有结构，不是堆字数
- **约束比自由更重要**：对 Agent 来说，"不能做什么"往往比"要做什么"更能提升质量
- **Audience 是最容易被忽略的要素**：同样的分析，给投委会看和给实习生看，输出应该完全不同

### 09:50-10:20 三要素讲解：Context（30分钟）

#### 概念引入（5分钟）

**类比**：
> "Context 就是你给下属提供的'背景资料包'。给少了，他做决策时信息不足；给多了，他会迷失重点。更关键的是，Agent 有'失忆'问题。"

**核心问题**：
> "Agent 能'记住'多少？—— 这取决于它的**上下文窗口**（Context Window）。超出这个窗口，早期的信息会被'遗忘'。所以，**管理 Context 是管理 Agent 能力的关键**。

#### 四种 Context 类型（15分钟）

**类型1：外部 Context（你主动给 Agent 的文件/数据）**

> "这是最直接的 Context。你上传一份财报PDF，Agent 就能基于它分析。但这里有技巧——"

- **文件 vs 文本**：PDF 需要解析，可能有格式损失；纯文本更可控
- **信息密度**：给 Agent 一份100页的招股书，它可能迷失在细节中；先给一份5页的摘要，再按需深入
- **结构化数据**：CSV/Excel 比纯文本更容易让 Agent 做计算和对比

**现场演示**：
- 先不给行业背景，让 Agent 分析某家公司的商业模式 → 输出泛泛
- 补充一句话行业 Context（"这是2024年的新能源行业，面临产能过剩"）→ 输出立刻聚焦

**类型2：对话 Context（历史对话的累积）**

> "Agent 能'记得'你们之前说了什么。但这个记忆是有限的。"

- **长对话的陷阱**：聊了50轮后，Agent 可能"忘记"你第一轮给的任务目标
- **应对方法**：定期"重置 Context"——在关键节点重复核心目标；或开启新会话

**类型3：系统 Context（CLAUDE.md / AGENTS.md / GEMINI.md）**

> "这是最高级的 Context 管理方法。你可以写一个配置文件，告诉 Agent：'无论何时何地，你都要遵守这些规则。'"

- 在 Claude Code 中，项目根目录下放一个 `CLAUDE.md`，内容会被自动读取为系统提示词
- 可定义：Agent 的角色、工作风格、默认输出格式、常用工具、禁忌事项

**示例**（展示在屏幕上）：

```markdown
# CLAUDE.md

## 角色
你是某顶级投研机构的资深分析师，擅长从公开信息中提取投资洞察。

## 工作风格
- 数据驱动，每个结论都要有数据支撑
- 对不确定的信息明确标注[推测]
- 优先使用定量分析，定性分析作为补充

## 输出格式
- 财务数据用 Markdown 表格
- 核心洞察用 bullet points，每条不超过30字
- 最后必须附"风险提示"段落

## 禁忌
- 不要编造未公开的财务数据
- 不要给出具体的买入/卖出建议（我们只提供分析，决策由投委会做出）
```

> "有了这个文件，你不需要每次对话都重复这些要求。Agent 会'记住'它是谁、该怎么做。"

**类型4：工具 Context（MCP 给 Agent 的'手脚'）**

> "Agent 能调用哪些工具，决定了它能完成的任务范围。这就像你给下属配了什么样的团队——只有Excel的人做分析，和有数据团队的人做分析，产出完全不同。"

- MCP（Model Context Protocol）是 Anthropic 推出的开放标准
- 通过 MCP，Agent 可以访问本地文件系统、数据库、搜索引擎、GitHub 等
- 工具越多，Agent 的能力越强，但也需要更好的 Harness 来约束

#### 教学要点总结（10分钟）

- **Context 的质量 > 数量**：给 Agent 10份杂乱的材料，不如给1份整理好的摘要
- **上下文窗口是硬约束**：长任务要拆分成短会话，关键信息要反复提及
- **系统 Context 是杠杆**：一次写好 CLAUDE.md，终身受益
- **工具 Context 决定天花板**：MCP 让 Agent 从"聊天"变成"做事"

### 10:20-10:50 三要素讲解：Harness（30分钟）

#### 概念引入（5分钟）

**类比**：
> "Harness 就是公司的 SOP（标准作业程序）和风控红线。Agent 能力很强，但也容易'幻觉'——编造不存在的信息，或'跑偏'——偏离你的真实需求。Harness 就是给它画跑道。"

**核心问题**：
> "Agent 为什么需要 Harness？因为它太'自信'了。它不会说'我不知道'，而是会编一个看起来像答案的东西。这在投资分析中是致命的。"

#### Harness 的四种类型（15分钟）

**类型1：输出格式约束**

> "告诉 Agent'我要什么形式'，而不是让它自由发挥。"

- "用 Markdown 表格输出"
- "每条洞察不超过50字"
- "先给结论，再给论据"
- "分三个部分：现状、风险、建议"

**现场演示**：
- 无约束：Agent 输出一段流畅但无结构的文字
- 有约束：Agent 输出清晰的表格+要点+风险提示

**类型2：思维链约束**

> "要求 Agent'先想后说'，而不是直觉反应。"

- "先列出你的分析框架，再逐条填充"
- "对每个结论，说明你的推理过程和依据来源"
- "在给出最终建议前，先列出所有可能的风险因素"

**教学要点**：
> "思维链约束不仅是为了输出质量，更是为了**可审计**——你可以检查 Agent 的推理过程，发现它在哪一步出了问题。"

**类型3：边界约束**

> "明确告诉 Agent'你不能做什么'。"

- "只基于上传的 PDF 内容分析，不要补充外部知识"
- "如果某项数据在材料中没有明确提及，标注[材料中未提及]"
- "不要给出具体的投资建议（买入/卖出/持有），只提供分析"

**现场演示"翻车教学点"**：
- 让 Agent 分析一份2023年的财报，但问它"2024年的业绩预期如何"
- **预期结果**：Agent 可能基于训练数据编造2024年数据，而不是说"材料中没有"
- **教学价值**：展示为什么边界约束是必要的

**类型4：校验机制**

> "让 Agent 自己检查自己。"

- "如果某条数据看起来异常（如毛利率超过100%），请标注[数据异常，需核实]"
- "完成分析后，请用一句话总结：'以上分析中，我最不确定的部分是...'"
- "请列出3个你可能遗漏的关键因素"

#### 教学要点总结（10分钟）

- **Harness 是安全带，不是枷锁**：好的 Harness 让 Agent 更快更准，不是更慢更死
- **防幻觉的三种方法**：
  1. 边界约束（只基于给定材料）
  2. 校验机制（自我检查）
  3. 人工复核（永远不要完全信任 Agent 的输出）
- **Harness 需要迭代**：没有一次完美的 Harness，要根据输出质量不断调整

### 11:05-11:35 工具链演示（30分钟）


**步骤2：CC Switch 配置演示（10分钟）**

```bash
claude

# 执行一个简单任务，确认响应正常
> 请用一句话介绍你自己
```

### 11:35-12:05 演示案例1：市场调研（30分钟）

#### 场景设定

> "假设中欧商学院计划开设一门'AI与产业变革'新课，我们需要快速了解同类课程的市场格局。"

#### 完整演示流程

**Prompt 设计（5分钟）**

展示在屏幕上，边写边讲解：

```
背景：中欧商学院计划开设一门面向高管的"AI与产业变革"短期课程，时长2天。

任务：调研全球顶尖商学院同类课程的设置情况，为课程设计提供参考。

目标商学院：哈佛商学院、斯坦福GSB、INSEAD、沃顿商学院、MIT斯隆

要求：
1. 对每家商学院，提取：课程名称、时长、核心模块、目标学员、公开价格（如有）
2. 用Markdown表格呈现对比结果
3. 基于对比，提炼3条差异化定位建议（中欧可以突出的独特价值）
4. 对无法确认的信息，标注[未确认]

约束：
- 优先使用各商学院官网的公开信息
- 不要编造课程价格或学员数据
- 如果某家商学院没有直接对应的课程，标注[无直接对应课程]
```

> "大家看这个 Prompt，它包含了 Context（中欧要开新课）、Objective（调研5家商学院）、Style（表格对比）、Audience（课程设计团队）、Constraint（不编造）。"

**Context 投喂（5分钟）**

- 方法1：让 Agent 直接联网搜索（如果配置了搜索 MCP）
- 方法2：提前准备好5家商学院的官网链接，作为 Context 输入
- 方法3：提前下载好相关网页的文本，作为文件上传

> "这三种方法的区别在于'可控性'。直接搜索最快，但 Agent 可能跑到无关页面；给链接次之；上传整理好的文本最慢但最可控。"

**Harness 约束（5分钟）**

在 Prompt 中已经包含了约束，但可以在执行过程中补充：

```
> 请确保每家商学院的信息都标注了信息来源（官网/新闻/其他）
> 如果发现某家商学院的信息自相矛盾，请列出矛盾点并标注[需核实]
```

**输出校验与迭代（10分钟）**

- 展示 Agent 的第一次输出
- 带领学员一起检查：有没有编造的价格？有没有遗漏的商学院？对比维度是否完整？
- 演示如何迭代："我注意到沃顿的信息似乎不完整，请补充"
- 展示第二次输出，对比改进

> "这个迭代过程是最重要的。第一次输出很少完美，但通过人工校验 + Prompt 调整，第三次通常就可用。"

#### 教学总结（5分钟）

- 这个案例展示了"信息收集 → 结构化对比 → 洞察提炼"的完整链条
- 强调：Agent 擅长"收集和整理"，但"差异化定位建议"需要人的判断
- 预告下午小组实战：每组会选择类似的市场调研任务，但换成自己的行业

### 12:05-12:35 演示案例2：财报快速解读（30分钟）

#### 场景设定

> "假设我们是某私募基金的投研团队，需要快速评估宁德时代2024年Q3的投资价值。"

#### 完整演示流程

**材料准备（5分钟）**

- 提前准备好宁德时代的财报 PDF（或文本摘要版）
- 上传给 Agent 作为外部 Context

**Prompt 设计（5分钟）**

```
背景：我们是某私募基金投研团队，正在评估宁德时代（300750.SZ）2024年Q3的投资价值。

任务：基于上传的财报PDF，提取核心财务指标并进行初步分析。

要求：
1. 提取以下指标（2024年Q3及同比/环比变化）：
   - 营业收入、归母净利润
   - 毛利率、净利率
   - 经营性现金流净额
   - ROE（净资产收益率）
   - 资产负债率
2. 对每项指标，用一句话说明"这个数字说明什么"
3. 识别2个最积极的变化和2个最需要关注的风险
4. 输出格式：Markdown表格 + 要点列表

约束：
- 只基于上传的PDF内容，不要补充外部信息
- 如果某项指标在PDF中没有明确披露，标注[PDF中未直接披露]
- 不要给出买入/卖出建议，只提供分析
```

**重点教学：翻车演示（10分钟）**

这是本案例的核心教学价值。

**故意不设边界约束**，让 Agent 自由发挥：

```
> 请分析宁德时代2024年Q3的财务表现
```

**预期翻车**：
- Agent 可能"自信地"编造一些数据
- Agent 可能基于训练数据补充2024年Q4的"预期"
- Agent 可能给出具体的投资建议（"建议买入"）

**带领学员发现错误**：
> "大家看这里，Agent 说'预计Q4营收增长15%'——PDF里有这个预测吗？没有。它'幻觉'了。这就是为什么边界约束和校验机制是必须的。"

**演示如何纠正**：
- 补充边界约束："只基于PDF，不要预测未来"
- 补充校验机制："完成后请列出你最不确定的3个数据点"
- 重新执行，对比两次输出

**人工复核教学（5分钟）**

- 展示 Agent 提取的财务数据
- 带领学员和原始 PDF 核对一两个关键数字
- 强调："Agent 可能把'亿'看成'万'，可能把'同比增长'理解成'环比增长'——**最终的数据准确性，必须由人负责。**"

#### 教学总结（5分钟）

- **财报分析是最典型的"高价值高风险"场景**：Agent 能极大提速，但一个数字错误就可能导致错误决策
- **防幻觉三板斧**：
  1. 边界约束（只基于给定材料）
  2. 校验机制（自我怀疑）
  3. 人工复核（最终守门）
- **Agent 的价值不是"替代分析师"，而是"让分析师把精力放在判断上，而不是搬运数据上"**

### 17:15-17:30 总结与点评（15分钟）

#### 共性问题总结（5分钟）

基于下午巡视和汇报，提炼3-5个共性问题：

> "今天我观察到一个有趣的现象：80%的小组在第一次 Prompt 里都忘了写 Audience。这导致 Agent 的输出要么太学术，要么太口语化。记住，**同样的分析，给投委会看和给实习生看，应该是完全不同的**。
>
> 第二个共性问题是 Harness 偏弱。很多组的 Prompt 很详细，但没有明确的'边界约束'和'校验机制'。结果是 Agent 很自信地给出了一些... 嗯，' creatively accurate' 的数据。"

#### 核心认知提炼（5分钟）

用一张图总结全天：

```
驾驭 Agent = 设计好任务（Prompt）+ 管理好信息（Context）+ 设定好规则（Harness）

        Prompt（任务书）
              ↓
    Context（背景资料） → Agent 执行 → Output（输出）
              ↑
        Harness（约束护栏）

最终守门人：人（你）
```

**三句话带走**：

1. **Agent 是杠杆，不是替代**：它放大你的执行力，但不替代你的判断力
2. **质量 80% 取决于你的 Prompt，15% 取决于 Context，5% 取决于模型**：不要迷信更贵的模型，先优化你的指令
3. **永远假设 Agent 会犯错**：边界约束、校验机制、人工复核——三道防线缺一不可

## 附录：讲师快速参考

### 常见翻车场景及应对

| 翻车场景 | 表现 | 应对话术 |
|---------|------|---------|
| Agent 编造数据 | 给出精确但不存在的数字 | "大家注意，Agent 不'知道'自己不知道。这就是为什么要加边界约束：'只基于给定材料'。" |
| Agent 跑偏 | 输出和任务无关 | "Prompt 的目标不够聚焦。试试把 Objective 拆成更具体的子任务。" |
| Agent 输出太长 | 信息淹没在长篇大论中 | "给 Agent 格式约束：'每条不超过30字'、'用表格呈现'。" |
| Agent 遗漏关键信息 | 没提取出用户关心的指标 | "Context 投喂不够精准。试试先把材料做一层人工筛选，再给 Agent。" |
| Agent 语气不对 | 太学术或太口语 | "Prompt 里加 Audience 和 Style：'面向投委会'、'资深分析师口吻'。" |
| 网络/API 故障 | cc-switch 切换失败 | "这正是 cc-switch 的价值——切换到备用 Provider。现场演示。" |

### 时间应急方案

| 情况 | 调整方案 |
|------|---------|
| 上午某模块超时 | 压缩演示案例的时间，保留三要素讲解 |
| 下午实战时间不够 | 取消休息，或缩短汇报到每组4分钟 |
| 某组进度严重滞后 | 助教一对一帮扶，或建议缩小任务范围 |
| 多个学员环境未配置好 | 让这些学员和已完成的同学组队，课后单独辅导 |

### 情绪管理

- **如果学员对 Agent 输出失望**："第一次失望是正常的。Agent 不是魔法，它需要你的引导。试着调整 Prompt，你会发现第二次好很多。"
- **如果学员过于兴奋，想完全依赖 Agent**："Agent 可以帮你写初稿，但签字的是你。最终的责任，永远在人的肩上。"
- **如果学员纠结于技术细节（如模型原理）**："这些细节很重要，但今天我们的重点是'怎么用'。课后我们可以单独讨论技术原理。"
