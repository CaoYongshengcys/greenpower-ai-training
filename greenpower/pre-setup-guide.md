## 写在前面：为什么专业人士需要掌握 AI Agent

作为绿色能源领域的从业者，你已经具备扎实的行业知识。本培训的目标不是重复你已经懂的内容，而是提供一套**将专业知识转化为自动化工作流**的方法论。

AI Agent 与普通对话模型的核心区别在于**工具调用与自主执行**。它不只是一个能回答问题的聊天机器人，而是能够：
- 读取本地文件系统（PDF 政策文档、Excel 数据集、项目代码）
- 执行终端命令（数据分析、格式转换、批量处理）
- 调用外部工具（网页抓取、数据库查询、API 交互）
- 在多步骤任务中保持上下文一致性

**本次培训的实战目标**：工作坊结束时，你能基于自身业务场景，独立构建一个可复用的 AI Agent 工作流（例如：自动分析政策文件并生成结构化摘要，或处理电站运行数据并输出异常诊断报告）。

## 第一阶段：Prompt —— 从指令到工程化

> **本阶段目标**：掌握 Prompt Engineering 的核心方法论，理解如何从"提问"升级为"定义任务规范"。

### 1.1 Prompt 的本质

Prompt 是人与 AI Agent 之间的**任务接口定义**。一个高质量的 Prompt 需要同时回答三个问题：

1. **角色与知识边界**：AI 应以什么身份执行？（新能源政策研究员 / 电力系统调度工程师 / 碳资产管理师）
2. **任务规范**：具体做什么？输入是什么？输出应满足什么条件？
3. **输出格式**：结果以什么结构呈现？是否需要特定模板或数据格式？

### 1.2 Prompt 工程三要素与进阶技巧

**基础框架（三要素）**：

| 要素 | 定义 | 示例 |
|------|------|------|
| **角色设定** | 定义 AI 的专业身份与知识域 | "你是一位从事电力市场分析 8 年的研究员" |
| **任务描述** | 明确输入、处理逻辑、边界条件 | "分析 2020-2025 年山东/河南/河北分布式光伏装机增速差异，并总结原因" |
| **输出格式** | 规定结果的结构、单位、呈现方式 | "输出 Markdown 表格：年份、全国累计装机(GW)、增速(%)、领先省份" |

**进阶技巧**：

- **Few-shot 示例**：在 Prompt 中嵌入 1-2 个输入-输出示例，引导模型理解预期格式
- **Chain-of-Thought**：要求模型先展示推理过程，再给出结论（适用于复杂分析任务）
- **Schema 约束**：用 JSON/Markdown 表格等结构化格式强制规范输出，便于下游程序解析
- **约束层叠**：先定义"必须包含的字段"，再定义"可选补充信息"，降低输出方差

### 1.3 绿色能源场景 Prompt 示例

**场景：政策文档条款提取**

```
你是一位新能源政策合规分析师。请分析附件《新型电力系统发展蓝皮书》中
与"分布式光伏参与电力市场交易"相关的全部条款。

任务要求：
1. 提取准入条件、交易品种、结算规则三类信息
2. 标注每条信息的原文页码和章节
3. 对模糊条款给出你的专业解读

输出格式：Markdown 表格，列：条款类型 | 原文摘要 | 页码 | 解读
```

**场景：发电数据异常诊断**

```
你是一位光伏电站性能分析师。附件为某 50MW 光伏电站 2024 年 7 月的
实际发电功率与气象数据（辐照度、组件温度）。

任务要求：
1. 识别发电量显著低于预期的日期（偏差 > 15%）
2. 分析异常日的气象与运行数据，给出最可能的技术原因
3. 提出运维建议

输出格式：先给出异常日期列表，再逐日分析，最后汇总建议。使用专业术语。
```

## 第二阶段：Context —— 构建 AI 的认知基座

> **本阶段目标**：理解 Context 在 Agent 工作流中的技术机制，掌握文件投喂、项目上下文和外部知识接入的方法。

### 2.1 Context 的技术定义

在 Claude Code 等 Agent 环境中，Context 包含三层：

| 层级 | 来源 | 机制 | 适用场景 |
|------|------|------|---------|
| **对话上下文** | 用户输入 | 自动维护在多轮对话中 | 澄清、追问、迭代优化 |
| **文件上下文** | 用户上传/项目目录 | 通过 `@file` 引用或自动扫描项目文件 | 分析政策 PDF、处理数据 Excel、读取代码库 |
| **知识库上下文** | 外部向量数据库/RAG | 通过 Skill 或 API 接入 | 企业私有知识库、行业标准库、历史项目档案 |

**关键约束**：单次请求的 Context 存在 Token 上限（通常 8K-200K，取决于模型）。超长文档需要分段或先提取关键章节。

### 2.2 绿色能源场景的 Context 策略

**策略 1：长文档分段处理**
- 先让 Agent 读取目录和摘要，定位相关章节
- 再针对目标章节发起深度分析请求
- 避免一次性上传 200 页 PDF 导致信息淹没

**策略 2：多源数据整合**
- 将政策文件（PDF）+ 企业数据（Excel）+ 项目参数（文本）同时作为 Context
- 在 Prompt 中明确各文件的用途（"附件 A 为政策原文，附件 B 为企业 2024 年能耗数据"）

**策略 3：项目级上下文感知**
- 在 Claude Code 中打开项目文件夹后，Agent 会自动读取目录结构、代码文件、配置文件
- 适用于构建自动化数据处理脚本或报告生成工具

## 第三阶段：Harness —— 工具链部署与配置

> **目标**：完成 Claude Code + CC Switch + 阿里百炼的完整部署，验证连通性。
>
> **以下步骤假设你具备基础的终端操作经验。**

### 3.1 环境准备

**系统要求**：
- macOS 12+ / Windows 10+（需 WSL2）/ Linux（Ubuntu 22.04+）
- 内存 ≥ 8GB，磁盘空间 ≥ 2GB
- 可访问互联网（安装阶段需下载 npm 包）

**检查 Node.js**：

```bash
node --version   # 需 ≥ v18
```

若未安装或版本过低：
- **macOS**: `brew install node`
- **Linux**: `curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt-get install -y nodejs`
- **Windows**: 在 WSL2 Ubuntu 终端中执行上述 Linux 命令

### 3.2 安装 Claude Code

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

**Windows 用户**：Claude Code 依赖 Linux 环境，请在 WSL2 Ubuntu 终端中执行上述命令。

**首次运行**：
```bash
claude
```
若计划通过 CC Switch 切换至阿里百炼，可跳过 Anthropic 官方登录，待配置完成后再重新启动。

### 3.3 安装 CC Switch

CC Switch 是 API 供应商路由管理工具，用于在国内网络环境下将 Claude Code 的后端从 Anthropic 官方 API 切换至阿里百炼。

**macOS**（推荐）：
```bash
brew tap farion1231/ccswitch
brew install --cask cc-switch
```
> 若 Homebrew 失败，可手动下载：[GitHub Releases](https://github.com/farion1231/cc-switch/releases)

**Windows**：下载 `.msi` 安装包（同上 Releases 页面）

**Linux**：下载对应发行版的 `.deb`/`.rpm`/`.AppImage`

**首次启动**：打开 CC Switch → 左侧选择 "Claude Code" → "添加供应商" → 保留默认，待下一步配置阿里百炼。

### 3.4 申请阿里百炼 API 账号

1. 访问 [阿里百炼控制台](https://www.alibabacloud.com/help/zh/model-studio/getting-started/what-is-model-studio)
2. 使用阿里云账号登录（无账号需先注册）
3. 开通"模型服务"（新用户通常有免费试用额度）
4. 进入 "API Key 管理" → "创建 API Key" → 复制以 `sk-` 开头的 Key

> **安全提示**：API Key 等同于访问凭证。请勿提交至 Git 仓库、截图外传或写入公开文档。本次配置通过 CC Switch 本地管理，不会暴露在命令行历史中。

### 3.5 配置供应商并验证

**CC Switch 配置**：
1. 打开 CC Switch → "Claude Code" 标签
2. 添加供应商 → 选择"自定义"
3. 填写：
   - **名称**：阿里百炼
   - **API Key**：粘贴复制的 Key
   - **Base URL**：`https://dashscope.aliyuncs.com/compatible-mode/v1`
4. 保存并设为默认供应商

**连通性验证**：
```bash
claude
```
进入交互界面（显示 `>` 提示符）后，输入测试 Prompt：
```
请用一句话定义"新型电力系统"，并说明其与传统电力系统的核心差异。
```
收到合理回复即表示 Harness 部署完成。

### 3.6 安装 Skills（可选扩展）

Skills 是 Claude Code 的声明式工作流扩展，通过 Markdown 配置文件定义触发条件、执行逻辑和输出规范。对于绿色能源专业工作流，以下通用 Skills 具有较高实用价值：

| Skill | 功能 | 专业场景 |
|-------|------|---------|
| **docx** | Word 文档生成与解析 | 自动生成碳盘查报告、政策合规审查文档 |
| **xlsx** | Excel 数据处理 | 发电量统计分析、财务模型计算、批量数据清洗 |
| **pdf** | PDF 文本提取与摘要 | 政策文件结构化解析、长文档关键信息抽取 |
| **firecrawl** | 网页抓取与解析 | 追踪政府官网政策更新、获取实时电价/补贴信息 |

**安装方法**：
```bash
mkdir -p ~/.claude/skills
# 将 Skill 文件夹（如 docx/SKILL.md）复制至上述目录
```

验证：在 Claude Code 中输入 `/ls`，确认已加载的 Skills 列表。

## 常见问题（FAQ）

### Q1: 阿里百炼与 Anthropic 官方 API 的模型能力差异？

阿里百炼通过兼容模式提供通义千问系列模型（如 qwen-max、qwen-plus）。在代码理解、长文本处理和中文语义理解方面表现良好，可作为 Claude Code 的可靠后端。若需对比特定任务的效果，可在 CC Switch 中快速切换供应商进行 A/B 测试。

### Q2: 长文档处理时的 Token 限制与分段策略？

当前主流模型的上下文窗口为 8K-200K Token（约 6K-150K 汉字）。处理超长政策文件（如 200 页蓝皮书）时，建议：
1. **目录导航**：先让 Agent 读取目录，定位至相关章节
2. **分段提取**：按章节或页码范围分批上传，避免信息淹没
3. **摘要前置**：先要求生成全文摘要，再基于摘要决定深入分析的章节

### Q3: 企业敏感数据上传至云端 API 的合规风险？

阿里百炼作为阿里云产品，数据处理和存储符合中国网络安全及数据安全法规。对于高敏感数据（如电网运行核心参数、未公开的项目财务数据），建议：
1. 在工作坊中使用**脱敏样本数据**
2. 课后评估企业内部部署方案（如私有化模型或本地 RAG 架构）
3. 与合规部门确认数据分级和跨境传输政策

### Q4: 如何评估 AI Agent 输出结果的专业可靠性？

AI Agent 的输出应被视为**初稿或辅助分析**，而非最终结论。建议建立以下校验机制：
1. **来源追溯**：要求 Agent 标注结论依据的原文页码或数据行号
2. **交叉验证**：对关键数据点，使用传统工具（Excel/Python）独立复算
3. **专家评审**：将 Agent 生成的分析报告提交给领域专家进行终审
4. **版本控制**：保留 Prompt 和原始输出，便于审计和迭代优化

### Q5: 安装 Claude Code 时 npm 报权限错误？

```bash
# 方案 A：使用 npx 临时运行（无需全局安装）
npx @anthropic-ai/claude-code

# 方案 B：修正 npm 全局目录权限
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH
```

### Q6: CC Switch 无法切换供应商或界面异常？

1. 重启 CC Switch
2. 检查 API Key 是否以 `sk-` 开头，且复制完整（无多余空格）
3. 确认 Base URL 准确：`https://dashscope.aliyuncs.com/compatible-mode/v1`
4. 查看 [GitHub Issues](https://github.com/farion1231/cc-switch/issues) 获取已知问题排查方案

### Q7: 阿里百炼 API Key 显示额度为 0？

- 检查阿里云账号是否已完成**实名认证**（通常需支付宝扫码，几分钟完成）
- 确认是否已开通"模型服务"而非仅创建 Key
- 免费额度用尽后可充值少量金额（通常数元即可满足工作坊需求）
